# 指标体系：PRD Copilot

## 1. 北极星指标

| 指标 | 定义 | 原因 |
| --- | --- | --- |
| 有效 PRD 生成率 | 用户输入需求后，成功生成并复制或导出 PRD 的比例 | 直接反映工具是否完成“模糊需求到 PRD”的核心价值 |

## 2. 核心漏斗

```text
访问页面
  → 输入需求
  → 点击生成
  → 查看澄清结果
  → 查看 PRD
  → 复制 / 导出
```

| 环节 | 指标 |
| --- | --- |
| 访问 | 页面访问量 |
| 输入 | 需求输入率 |
| 生成 | 生成触发率 |
| 澄清 | 澄清结果查看率 |
| PRD | PRD 查看率 |
| 采纳 | 复制率 / 导出率 |

## 3. 质量指标

| 指标 | 定义 |
| --- | --- |
| 平均需求成熟度 | 所有生成需求的平均成熟度分数 |
| 缺失项平均数量 | 每次分析识别出的缺失 PRD 要素数量 |
| 补充后分数提升 | 用户补充信息后，成熟度分数提升幅度 |
| PRD 完整度 | 生成文档中包含必备章节的比例 |
| 评审一次通过率 | 使用该工具产出的 PRD 在人工评审中一次通过的比例 |

## 4. 体验指标

| 指标 | 目标 |
| --- | --- |
| 生成耗时 | 2 秒内 |
| 空输入错误率 | 越低越好 |
| 复制成功率 | 98% 以上 |
| 移动端可读性 | 主要内容无遮挡 |
| 用户满意度 | 持续提升 |

## 5. 埋点方案

| 事件名 | 触发时机 | 属性 |
| --- | --- | --- |
| page_view | 页面打开 | device_type |
| requirement_input | 用户输入需求 | text_length、domain |
| generate_clicked | 点击生成 | domain、stage |
| clarification_generated | 生成澄清结果 | score、missing_count |
| prd_generated | 生成 PRD | section_count |
| prd_copied | 点击复制 | score、domain |
| prd_downloaded | 点击导出 | score、domain |
| sample_loaded | 切换样例 | sample_label |

## 6. 数据驱动迭代

```text
收集用户输入
  → 分析高频缺失项
  → 优化追问问题
  → 调整 PRD 模板
  → 提升生成后复制 / 导出率
```

如果大量用户缺少“指标”和“验收标准”，下一版应强化指标推荐和验收样例。
