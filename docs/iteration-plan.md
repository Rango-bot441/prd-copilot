# 迭代计划：PRD Copilot

## 1. 迭代原则

- 先验证“需求澄清 + PRD 生成”闭环。
- 再增强真实产品经理工作流。
- 最后接入模型、历史记录和团队协作。

## 2. 版本路线图

| 版本 | 目标 | 核心功能 |
| --- | --- | --- |
| MVP v1.0 | 完成作品集原型 | 需求输入、成熟度评分、追问、需求卡、PRD 生成 |
| v1.1 | 提升 PRD 质量 | 更细的评分规则、更多业务场景、PRD 缺口检查 |
| v1.2 | 支持多轮澄清 | 用户逐题回答，动态提升成熟度 |
| v2.0 | 接入大模型 | 自然语言追问、语义理解、个性化 PRD |
| v2.1 | 支持 PRD Review | 粘贴已有 PRD，输出问题和优化建议 |
| v3.0 | 团队协作 | 历史版本、评审状态、评论和导出 |

## 3. MVP v1.0

已实现：

- 三栏式工作台。
- 模糊需求输入。
- 业务场景选择。
- 需求成熟度评分。
- 缺失信息识别。
- 建议追问。
- 需求卡。
- PRD 生成。
- 复制和导出 Markdown。
- 作品集 README 和产品文档。

## 4. v1.1：PRD 质量增强

| 功能 | 价值 |
| --- | --- |
| 更细的评分模型 | 区分用户、场景、目标、流程、指标、风险 |
| PRD 缺口检查 | 明确哪些章节不够完整 |
| 更多行业模板 | 支持内容、电商、教育、企业效率等 |
| 示例库 | 帮用户学习好需求和坏需求的差异 |

## 5. v1.2：多轮澄清

用户可以逐个回答系统追问，系统动态更新：

- 需求成熟度。
- 需求卡。
- PRD 草稿。
- 待澄清问题。

验收标准：

- 用户每补充一个回答，分数和 PRD 内容自动更新。
- 支持保存本轮澄清结果到本地。

## 6. v2.0：AI 能力接入

接入大模型后，能力升级为：

- 语义级需求分析。
- 更自然的追问。
- 更贴业务的 PRD。
- 支持不同角色视角：产品、研发、设计、运营、数据。

安全要求：

- API Key 必须放在服务端。
- 用户输入需要隐私提示。
- 输出需要标注“AI 草稿，需人工确认”。

## 7. v2.1：PRD Review Bot

新增模式：

```text
粘贴已有 PRD
  → 系统检查缺失章节
  → 输出风险和修改建议
  → 生成评审摘要
```

价值：

- 更贴近真实 PRD 评审场景。
- 证明候选人不仅会写 PRD，也会审 PRD。

## 8. v3.0：团队协作

长期方向：

- 登录和项目空间。
- PRD 历史版本。
- 评审状态流转。
- 评论和批注。
- 导出为 Markdown / PDF / Docx。

## 9. 求职作品集优先级

如果用于面试展示，建议优先做：

1. PRD Review Bot。
2. 多轮澄清。
3. 示例库。
4. 生成结果截图和 README 优化。

这些功能最能证明 PRD 能力、产品判断和跨团队评审意识。
