# 原型说明：PRD Copilot

## 1. 原型定位

PRD Copilot 是一个三栏式产品工作台，用来展示从模糊需求到 PRD 草稿的完整过程。

## 2. 页面结构

```text
PRD Copilot
├── 输入模糊需求
│   ├── 一句话需求
│   ├── 业务场景
│   ├── 需求阶段
│   ├── 补充信息
│   ├── 生成按钮
│   └── 样例切换
├── 需求澄清 Copilot
│   ├── 需求成熟度评分
│   ├── 已识别要素
│   ├── 缺失信息
│   ├── 建议追问
│   └── 需求卡
└── PRD Builder
    ├── PRD 预览
    ├── 复制 PRD
    └── 导出 Markdown
```

## 3. 核心交互

### 3.1 输入需求

用户输入一句模糊需求，并选择业务场景和需求阶段。

设计目的：

- 保留真实工作中的模糊起点。
- 用业务场景帮助系统给出更贴近的用户、痛点和指标。

### 3.2 生成澄清结果

系统输出：

- 需求成熟度评分。
- 已识别 PRD 要素。
- 缺失信息。
- 建议追问。
- 需求卡。

设计目的：

- 让用户看到 PRD 不是直接生成，而是先澄清。

### 3.3 生成 PRD

右侧同步生成结构化 PRD。

章节包括：

- 背景与问题。
- 目标用户。
- 用户场景。
- 产品目标。
- MVP 功能范围。
- 用户流程。
- 功能需求。
- 指标体系。
- 风险与边界。
- 验收标准。
- 待澄清问题。

### 3.4 复制与导出

用户可以复制 PRD 或导出 Markdown。

设计目的：

- 让输出可以进入真实评审或 GitHub 文档。

## 4. 状态设计

| 状态 | 触发条件 | 页面反馈 |
| --- | --- | --- |
| 初始状态 | 首次进入 | 空输入和空结果提示 |
| 空输入 | 清空需求后生成 | 提示先输入需求 |
| 生成完成 | 输入有效需求 | 三栏同步更新 |
| 复制成功 | 点击复制 PRD | 按钮显示“已复制” |
| 导出成功 | 点击导出 MD | 下载 Markdown 文件 |

## 5. 样例设计

| 样例 | 目的 |
| --- | --- |
| 创作者选题 | 对应 AI 产品运营和创作者成长场景 |
| 内容安全 | 对应平台合规和发布前检测场景 |
| 数据复盘 | 对应指标体系和数据驱动迭代 |
| 运营提效 | 对应需求池和跨团队协作 |

## 6. 原型边界

- 当前不接入真实 AI。
- 当前不支持多轮对话。
- 当前不保存用户历史。
- 当前 PRD 为草稿，需要人工继续完善。
