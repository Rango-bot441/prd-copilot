# PRD：PRD Copilot

## 1. 文档信息

| 项目 | 内容 |
| --- | --- |
| 产品名称 | PRD Copilot |
| 当前版本 | MVP v1.0 |
| 产品类型 | 需求澄清与 PRD 生成工具 |
| 目标用户 | 产品经理、产品运营、求职作品集展示者 |
| 技术形态 | 纯前端静态 Web 应用 |

## 2. 背景与问题

很多产品需求最开始都只有一句模糊描述，例如“做一个 AI 选题推荐功能”或“做一个内容安全检测工具”。真正进入 PRD 之前，产品经理需要澄清用户是谁、问题是什么、目标是什么、第一版做什么、怎么判断有效、有哪些风险和边界。

如果直接把模糊需求写成 PRD，容易出现功能范围不清、指标缺失、验收标准不完整、研发无法估期、评审反复返工等问题。

PRD Copilot 试图把这个过程产品化：先判断需求成熟度，再指出缺失信息，最后生成一份可继续评审的 PRD 草稿。

## 3. 产品目标

- 帮用户快速判断一个需求是否足够进入 PRD 阶段。
- 帮用户补齐 PRD 前置澄清问题。
- 帮用户生成结构化 PRD 草稿。
- 作为作品集项目，直观展示 PRD、需求分析、指标设计和风险意识。

## 4. 目标用户

| 用户 | 需求 |
| --- | --- |
| 产品求职者 | 需要用作品集证明自己具备 PRD 能力 |
| 初级产品经理 | 面对模糊需求时，需要澄清框架 |
| 产品运营 | 希望把业务想法整理成产品可评审语言 |
| 创业者 / 独立开发者 | 希望把想法整理成可执行 MVP 文档 |

## 5. MVP 范围

### 包含

- 输入一句话需求。
- 选择业务场景和需求阶段。
- 填写补充信息。
- 生成需求成熟度评分。
- 识别缺失 PRD 要素。
- 生成建议追问问题。
- 生成需求卡。
- 生成 PRD 草稿。
- 支持复制 PRD 和导出 Markdown。

### 不包含

- 真实大模型 API。
- 多轮对话。
- 用户登录。
- 历史记录。
- 团队协作。
- 在线文档编辑。

## 6. 用户流程

```text
进入页面
  → 输入一句话需求
  → 选择业务场景和需求阶段
  → 填写补充信息
  → 点击生成澄清与 PRD
  → 查看需求成熟度
  → 查看缺失信息和追问问题
  → 查看需求卡
  → 查看 PRD 草稿
  → 复制或导出 Markdown
```

## 7. 功能需求

| 功能 | 描述 | 优先级 |
| --- | --- | --- |
| 需求输入 | 支持输入一句话需求和补充信息 | P0 |
| 场景选择 | 支持选择创作者工具、内容安全、数据分析等场景 | P0 |
| 成熟度评分 | 根据目标用户、痛点、目标、范围、指标、风险、验收标准计算分数 | P0 |
| 缺失信息识别 | 列出当前需求缺少的 PRD 要素 | P0 |
| 追问生成 | 按缺失项生成产品经理继续追问的问题 | P0 |
| 需求卡生成 | 输出用户、问题、目标、MVP、核心指标 | P0 |
| PRD 生成 | 输出结构化 PRD 草稿 | P0 |
| 复制 PRD | 一键复制 Markdown 文档 | P1 |
| 导出 Markdown | 下载 PRD 文件 | P1 |
| 样例切换 | 提供多个业务样例，方便演示 | P1 |

## 8. 指标体系

| 指标 | 定义 |
| --- | --- |
| 需求生成完成率 | 输入需求后成功生成 PRD 的比例 |
| PRD 复制率 | 用户生成后点击复制 PRD 的比例 |
| Markdown 导出率 | 用户生成后导出文档的比例 |
| 需求成熟度提升率 | 用户补充信息后成熟度分数提升的比例 |
| 评审一次通过率 | 使用工具产出的 PRD 在评审中一次通过的比例 |

## 9. 风险与边界

- 当前评分基于规则，不等同于真实产品专家判断。
- 当前 PRD 是草稿，不应直接替代人工产品设计。
- 模糊输入会导致目标用户、指标和风险推断不准。
- 如果接入真实模型，需要增加隐私保护和输入内容安全策略。

## 10. 验收标准

- 空输入时展示明确提示。
- 输入需求后能生成成熟度评分。
- 至少输出 3 类缺失信息或确认信息完整。
- 至少输出 5 个追问问题。
- PRD 草稿包含背景、用户、目标、范围、流程、指标、风险和验收标准。
- 复制 PRD 和导出 Markdown 可用。
- 移动端布局可正常阅读。
